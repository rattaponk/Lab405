Development workflow with Webpack and Vue CLI

Development Server

  • Vue runs on the Client
  • the App will be served by a server
  • file:// protocol is not the http:// protocol
  • test your App under realistic circumstances

Development Workflow

  • code -> apply "special features" -> production server
  • special features:
    • compile 'single file template' (don't use "template"!)
    • case-insensitive component selectors
    • preprocessors and more

Using Vue CLI to create projects

  • Vue CLI help fetching Vue project template
  • install Vue CLI using the command: npm install -g vue-cli
  • choose from different available template:
    • Simple: index.html + Vue CDN import
    • webpack-simple: basic Webpack workflow
    • webpack: complex Webpack workflow (incl. testing)
    • browserify / browserify-simple: browserify workflows

Installing the Vue CLI and creating a new project

  • https://github.com/vuejs/vue-cli
vue init <template-name> <project-name>

vue init webpack my-project

cd my-project

npm install

npm run dev

Webpack Template folder Structure

  • src is the most important
  • .babelrc: for writing ES6 and translate to ES5
  • index.html: template

Understanding ".vue" Files

  • render property specify a compiled vue template
  • the following code loads 'App.vue' file, another Vue app
\\ main.js file
import Vue from 'vue'
import App from './App.vue'

new Vue( {
  el: '#app',
  render: h => h(App)   // ES6 arrow function
 } )

// App.vue file - Single File Template

// HTML template, can have vue interpolation and ...
<template>
  <h1> Hello World! <h1>
</template>

// behave like a Vue instance : new Vue( { } )
<script>
  export default {
  }
</script>

// CSS style
<style>

</style>

How to build your App for production

  • build for production with the command: npm run build*
  • dist folder: stores your bundled and optimization (minify)

Debugging VueJs Project

When working with a Project created by the CLI (using Webpack), you can asily debug your running App by opening your Developer Tools and going to "Sources" tab.

  • Developer Tools in Chrome
  • Vue Developer Tools (https://github.com/vuejs/vue-devtools)
